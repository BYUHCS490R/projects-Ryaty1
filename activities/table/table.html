Answers the Questions

  1. When do you use a table?

    You use a table when you want to display data in a tabular format.

  2. When do you not use a table?

    Do not use a table for page layout or non-tabular content; prefer semantic elements and CSS for layout.

  3. When do you need to do extra work for table accessibility?

    When a table is complex (merged cells, multi-level headers), add captions, proper scope/headers/id attributes, and consider summaries for screen readers.
